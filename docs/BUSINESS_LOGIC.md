# Business Logic & App Rules

Este documento contiene las reglas de negocio principales para la aplicación Airline.

## 1. Objetivo General
Que los usuarios puedan adquirir aviones y licencias para poder efectuar vuelos. 

## 2. Gestión de Vuelos
- **Creador:** Los vuelos los crea `weifly`.
- **Recompensa:** Se ofrece una recompensa a quien efectue el vuelo.
- **Sincronización:** Cada vuelo ha de estar perfectamente sincronizado y validado con un **callsign** que se le asignará al usuario para que lo use en su simulador.
- **NFTs:** Cada vuelo debería generar un NFT (o un mecanismo similar de registro inmutable).

## 3. Combustible y Staking
Para poder volar hace falta gasolina.
- **Mecanismo:** Se obtiene poniendo en staking el token `AIRL`.
- **Ratio:** Produce **100 AIRG** al día por cada **1 AIRL** en staking.
